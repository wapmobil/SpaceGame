# Фаза 2: Core expedition chain engine (Go)

## Цель
Создать полностью рабочую систему expedition event chains на бэкенде. LLM генерация событий, обработка выборов, retry-логика, сохранение в БД.

## Что работает после деплоя
Методы `StartExpeditionChain()`, `ResolveChoice()`, `CompleteChain()`, `FailChain()` работают. API ещё нет.

## Задачи

### 7.1 Создать expedition_chain.go
**Файл:** `server/internal/game/planet_survey/expedition_chain.go` (NEW)

#### Структуры

```go
package planet_survey

import (
    "time"
)

// ExpeditionChain represents a running/completed expedition event chain.
type ExpeditionChain struct {
    ID                 string            // UUID
    PlanetID           string
    OwnerID            string
    Status             string            // "active" | "completed" | "failed"
    EventCount         int               // сколько событий сгенерировано
    CurrentEventIndex  int               // индекс следующего события для показа
    DiscoveredLocation *Location         // локация, обнаруженная в конце (nil если failed)
    Inventory          map[string]float64 // ресурсы экспедиции, суммарно <= 1000
    CreatedAt          time.Time
    UpdatedAt          time.Time
}

// ExpeditionEvent represents a single event generated by LLM.
type ExpeditionEvent struct {
    EventID           string
    Description       string
    ImmediateReward   map[string]float64 // flat: {"reagents": 30, "food": -10}
    Choices           []ExpeditionChoice
    IsEnd             bool
    LocationReward    string // тип локации, только при is_end: true
}

// ExpeditionChoice represents a player choice within an event.
type ExpeditionChoice struct {
    Label       string
    Description string
    Reward      map[string]float64 // flat: {"iron": 50, "composite": 20}
    NextEventID string
}
```

#### Константы

```go
const (
    MaxInventory    = 1000.0
    MaxLLMRetries   = 5
    PromptFile      = "expedition_event_prompt.txt"
)

// Допустимые ресурсы инвентаря (только 5)
var ValidInventoryResources = []string{"food", "iron", "composite", "mechanisms", "reagents"}
```

#### Функции

**`ValidateInventory(inventory map[string]float64) error`**
- Сумма всех ресурсов <= 1000
- Все значения >= 0
- Только ресурсы из `ValidInventoryResources` (food, iron, composite, mechanisms, reagents)
- Возвращает `PlanetError` с reason "invalid_inventory"

**`ReadPromptTemplate() (string, error)`**
- Читает `expedition_event_prompt.txt` из `server/internal/game/expedition/`
- Возвращает содержимое файла как string
- Используется для генерации промпта к LLM

**`BuildPrompt(planet *Planet, inventory map[string]float64, events []ExpeditionEvent) string`**
- Собирает JSON контекст:
  - planet.name, planet.description, planet.level, planet.ResourceType
  - inventory (текущие ресурсы экспедиции)
  - events (история событий — event_id, description, choices, player_choice, rewards_received)
  - expedition_depth = len(events)
  - max_depth = 6
- Читает prompt template
- Возвращает полный prompt для LLM (template + JSON context)

**`ParseReward(raw map[string]interface{}) (map[string]float64, error)`**
- Принимает raw JSON от LLM: `{"resources": {"reagents": 50, "food": -10}}`
- Извлекает `resources` → `map[string]float64`
- Фильтрует: оставляет только ресурсы из `ValidInventoryResources`
- Неизвестные ресурсы игнорируются (например, "diamond" от LLM)
- Возвращает flat map: `{"reagents": 50, "food": -10}`

**`ApplyRewardToInventory(inventory map[string]float64, reward map[string]float64)`**
- Для каждого ресурса в reward: `inventory[res] += amount`
- Clamp: если `inventory[res] < 0` → `inventory[res] = 0`
- Если `GetInventorySize(inventory) > MaxInventory` → пропорционально уменьшить до 1000
- Жёсткий clamp 1000 в любой момент

**`GetInventorySize(inventory map[string]float64) float64`**
- Сумма всех значений в inventory

**`ParseEventResponse(raw string) (*ExpeditionEvent, error)`**
- Принимает raw string от LLM
- Парсит JSON в ExpeditionEvent
- Валидирует: event_id не пустой, choices 2-4 шт (или 0 если is_end), is_end boolean
- Для `immediate_reward` и `choice.reward` — вызывает `ParseReward()` (вложенный формат → flat)
- Возвращает ошибку если JSON невалиден или поля не проходят валидацию
- Если LLM вернул markdown-обёртку (```json ... ```), убирает

**`GenerateEventWithRetry(prompt string) (*ExpeditionEvent, error)`**
- Вызывает LLM через `openai.New().Chat(ctx, prompt)`
- Ловит ошибку
- Retry до 5 раз с экспоненциальной задержкой: 1s, 2s, 4s, 8s, 16s
- После каждого вызова — `ParseEventResponse()`
- Если парсинг не удался после 5 ретраев — возвращает ошибку
- При успехе — возвращает `*ExpeditionEvent`

**`StartExpeditionChain(planet *Planet, inventory map[string]float64) (*ExpeditionChain, *ExpeditionEvent, error)`**
- Валидирует inventory (`ValidateInventory`)
- Создаёт `ExpeditionChain` с переданным inventory (ресурсы уже списаны с планеты)
- Генерирует промпт (`BuildPrompt` с пустыми events)
- Генерирует первое событие с retry (`GenerateEventWithRetry`)
- Применяет `immediate_reward` к inventory через `ApplyRewardToInventory`
- Сохраняет событие в chain (append к internal slice)
- Возвращает chain + первое событие

**`ResolveChoice(chain *ExpeditionChain, choiceIndex int) (*ExpeditionEvent, error)`**
- Валидирует choiceIndex (0 <= index < len(current_event.Choices))
- Применяет награду за выбор к `chain.Inventory` через `ApplyRewardToInventory`:
  - Записывает выбор в историю chain
  - Записывает полученные награды в историю
- Проверяет `is_end` текущего события:
  - Если `is_end == true`:
    - Если `location_reward != nil`: создаёт Location (выбирает type по location_reward, генерирует name, считает source amount)
    - Вызывает `CompleteChain(chain, location)`
    - Возвращает nil event + chain (completed)
  - Если `is_end == false`:
    - Генерирует следующее событие с retry
    - Применяет `immediate_reward` к inventory через `ApplyRewardToInventory`
    - Сохраняет в chain
    - Возвращает новое событие
- При ошибке LLM — retry 5 раз, затем `FailChain(chain)`

**`CompleteChain(chain *ExpeditionChain, location *Location)`**
- Устанавливает `chain.Status = "completed"`
- Устанавливает `chain.DiscoveredLocation = location`
- Обновляет `chain.UpdatedAt`

**`FailChain(chain *ExpeditionChain)`**
- Устанавливает `chain.Status = "failed"`
- Обновляет `chain.UpdatedAt`
- Инвентарь НЕ возвращается автоматически — это делает handler при вызове

**`ReturnInventoryToPlanet(planet *Planet, inventory map[string]float64)`**
- Возвращает ресурсы из inventory на планету
- Для каждого ресурса: `planet.Resources[res] += amount`

**`GetCurrentEvent(chain *ExpeditionChain) *ExpeditionEvent`**
- Возвращает событие по индексу `chain.CurrentEventIndex`
- Если индекс вне bounds — возвращает nil

**`GetEventHistory(chain *ExpeditionChain) []ExpeditionEvent`**
- Возвращает все события chain

```go
func loadExpeditionChains(p *Planet) error {
    if p.game == nil || p.game.db == nil {
        return nil
    }
    // SELECT * FROM expedition_chains WHERE planet_id = $1 AND status = 'active'
    rows, err := p.game.db.Query(`SELECT id, planet_id, owner_id, status, event_count, 
        current_event_index, inventory, created_at, updated_at 
        FROM expedition_chains WHERE planet_id = $1 AND status = 'active'`, p.ID)
    if err != nil || rows.Err() != nil {
        return err
    }
    defer rows.Close()
    
    chains := make([]*ExpeditionChain, 0)
    for rows.Next() {
        var chain ExpeditionChain
        // scan columns
        chains = append(chains, &chain)
    }
    p.ExpeditionChains = chains
    return nil
}
```
